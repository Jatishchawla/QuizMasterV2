Project Title: Quiz Master - Online Quiz Management System

Question Statement:
Design and implement a full-stack web application for online quiz management. The system should allow admins to create/manage subjects, chapters, quizzes, questions, and users. Users can attempt quizzes, view their results, and track their analytics.

Approach:
- Used a modular approach with a clear separation of backend (Flask) and frontend (Vue.js).
- Designed the database schema to support subjects, chapters, quizzes, questions, users, and scores.
- Implemented RESTful APIs for all CRUD operations and quiz attempts.
- Used JWT authentication for secure access and role-based authorization.
- Built a responsive frontend with Vue.js, supporting admin and user dashboards, analytics, and quiz interfaces.
- Added analytics and reporting features for both admins and users.

Frameworks and Libraries Used:
------------------------------
Backend:
- Python 3.x
- Flask
- Flask-RESTful
- Flask-JWT-Extended
- Flask-CORS
- SQLAlchemy (ORM)
- SQLite (Database)

Frontend:
- Vue.js 3
- Vue Router
- Axios
- Chart.js (for analytics and charts)
- Bootstrap 5 (for UI components)

Other:
- bcrypt / werkzeug.security (for password hashing)

Tables:
- users: id, email, password, username, full_name, qualification, dob, role, created_at, updated_at
- subjects: id, name, description
- chapters: id, subject_id (FK), name, description
- quizzes: id, chapter_id (FK), subject_id (FK), name, date_of_quiz, time_duration, remarks
- questions: id, quiz_id (FK), chapter_id (FK), subject_id (FK), question_statement, option1, option2, option3, option4, correct_option, marks, question_type
- scores: id, quiz_id (FK), user_id (FK), time_stamp_of_attempt, total_scored, user_answers

Relations:
- A subject has many chapters
- A chapter belongs to a subject and has many quizzes
- A quiz belongs to a chapter and subject, and has many questions and scores
- A question belongs to a quiz, chapter, and subject
- A user can have many scores (quiz attempts)
